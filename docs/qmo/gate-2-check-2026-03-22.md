## QMOが発見した不整合（Design Reviewerが見逃した点）

### ❌ AC-10 のブレークポイント値誤記

**問題**: AC-10「モバイル（**<1024px**）で1カラムに縦積みされる」は誤り。

**根拠**:
- FR-107-4: 「タブレット（768〜1023px）では左カラム320px固定の**2カラム**表示、モバイル（**< 768px**）では1カラム」
- Section 3.8ブレークポイント表: ≥1024px → 2カラム / 768〜1023px → **2カラム** / < 768px → 1カラム
- つまり768〜1023px（タブレット）は2カラムであり、1カラムになるのは< 768px

**影響**: AC-10の記述通りに実装すると、タブレット（例: 900px）でも1カラムになってしまう設計上の矛盾が生じる。

**修正箇所**:
- `AC-10`: `モバイル（<1024px）` → `モバイル（< 768px）`

### ⚠️ E-23 のテストブレークポイント値不整合

**問題**: E-23「モバイル (640px) 1列への変化」は、実際の境界（768px）と異なるviewportで確認している。

**根拠**: Section 3.8では< 768pxが1カラム境界。640pxは確かに1カラムになるが、境界値（767px）のテストがない。

**修正箇所**:
- E-23: `モバイル (640px)` → `モバイル (767px)` かつ「viewport 767px以下（<768px）で縦積みに変化すること」に修正し、境界値テストを明確化する。
