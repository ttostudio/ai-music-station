# UI 仕様書 — プレイリスト機能

## 概要

既存の TabBar ナビゲーション（radio / tracks / likes）に「プレイリスト」タブを追加。
既存のレイアウトシステム（MobileLayout / TabletLayout / DesktopLayout）に準拠する。

---

## 画面構成

### 新規画面・タブ

| 画面ID | 名称 | 説明 |
|--------|------|------|
| SCREEN-PL-01 | プレイリスト一覧 | タブ「プレイリスト」選択時のデフォルト画面 |
| SCREEN-PL-02 | プレイリスト詳細 | プレイリスト選択後のトラック一覧 |
| MODAL-PL-01 | プレイリスト作成モーダル | 新規作成フォーム |
| MODAL-PL-02 | プレイリスト編集モーダル | 名前・説明の編集 |
| MODAL-PL-03 | トラックをプレイリストに追加モーダル | トラック選択時に表示 |

### コンポーネント

| コンポーネント名 | 説明 |
|---------------|------|
| `PlaylistDetail` | 詳細画面コンテナ |
| `PlaylistHeader` | 戻るボタン・プレイリスト名・「すべて再生」ボタン |
| `PlaylistTrackItem` | 1件のトラック行（ドラッグハンドル・タイトル・削除ボタン） |
| `PlaylistTrackList` | ドラッグ&ドロップ対応トラック一覧 |
| `AddTracksButton` | 「+ トラックを追加」→ MODAL-PL-03 を開く |

### インタラクション

| 操作 | 動作 |
|------|------|
| [▶ すべて再生] | playlistMode=true で Player に渡してトラック配列をセット |
| トラック行タップ | そのトラックから再生開始 |
| [✕] ボタン | ConfirmDialog なしで即削除（DELETE API 呼び出し、リスト更新） |
| ドラッグ（≡ ハンドル） | ドラッグ&ドロップで順序変更 → PUT reorder API 呼び出し |
| [← 戻る] | SCREEN-PL-01 へ戻る |

## MODAL-PL-03: トラックをプレイリストに追加

### トリガー

- トラック一覧（SCREEN tracks タブ）のトラックカードの「+」ボタン
- プレイリスト詳細画面の「+ トラックを追加」ボタン
- お気に入り一覧（likes タブ）のトラックカードの「+」ボタン

### レイアウト

```
┌─────────────────────────────────┐
│ プレイリストに追加           [✕] │
│ "Summer Vibe"                   │
├─────────────────────────────────┤
│ ┌─────────────────────────────┐ │
│ │ ☑ マイプレイリスト (5曲)    │ │
│ └─────────────────────────────┘ │
│ ┌─────────────────────────────┐ │
│ │ □ 作業BGM (12曲)           │ │
│ └─────────────────────────────┘ │
│ ─────────────────────────────── │
│ [+ 新しいプレイリストを作成]      │
│                                 │
│                        [完了]   │
└─────────────────────────────────┘
```

**ロジック:**
- チェック済み = 既にこのトラックが含まれているプレイリスト（チェックを外すと削除）
- チェックを入れると即座に POST tracks API を呼び出す（保存ボタンなし）
- 「+ 新しいプレイリストを作成」→ MODAL-PL-01 をネストして開く

## Player コンポーネントへの変更

### 再生モード追加

```typescript
// 既存: channelId による radio ストリーム再生
// 追加: playlist モードによるトラック配列再生

type PlaybackMode = 'radio' | 'playlist';

interface PlaylistPlaybackState {
  mode: 'playlist';
  playlist: PlaylistDetail;
  currentIndex: number;
  tracks: PlaylistTrackItem[];
}
```

### UI 変更（playlist モード時）

- ラジオストリームの代わりに HTML Audio で直接 MP3 ファイルを再生
- 「前のトラック」「次のトラック」ボタンを表示
- プレイリスト名・現在のトラック位置（例: "2/5"）を表示
- トラック終了時に自動的に次のトラックへ進む

## レスポンシブデザイン

| ブレークポイント | 対応レイアウト |
|---------------|-------------|
| モバイル（〜767px） | 1カラム、フルスクリーンモーダル |
| タブレット（768px〜1023px） | 2カラム可、モーダルは中央配置 |
| デスクトップ（1024px〜） | サイドバーにプレイリスト一覧、メインエリアに詳細 |

## アクセシビリティ

| 要件 | 実装方法 |
|------|---------|
| ドラッグ&ドロップのキーボード代替 | 矢印キーによる順序変更（↑↓） |
| モーダルフォーカストラップ | モーダル open 時に内部要素にフォーカス |
| ARIA ラベル | プレイリストカードに aria-label="プレイリスト: {名前}" |
| 削除確認 | [✕] ボタンに aria-label="トラックを削除: {タイトル}" |

## エラー状態の UI

| エラー | 表示 |
|--------|------|
| API 失敗（作成・削除等） | トースト通知「操作に失敗しました。もう一度お試しください。」 |
| プレイリスト名重複（409） | フォームフィールド下に「同じ名前のプレイリストが既に存在します」 |
| 上限超過（422） | 「プレイリストのトラック数の上限（200件）に達しています」 |
| 空のプレイリスト再生 | 「トラックがありません。トラックを追加してください。」 |
