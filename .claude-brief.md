# Development Brief — AI Music Station

## Mission
**AI Music Station - ACE-Step v1.5 によるストリーミング音楽生成**

Mac mini 上で ACE-Step v1.5 を使い、リクエストに応じて音楽を生成し、ラジオステーションのようにストリーミング再生する Web アプリケーションを構築する。

### Requirements
- チャンネル毎のリクエスト（例: アニソンチャンネル、LoFi チャンネル）
- 音楽生成 → キュー → ストリーミング配信
- ブラウザで再生可能（Tailscale 経由）
- Docker Compose で完結
- AI Company OS ダッシュボードから Start/Stop 可能

## Architecture

```
Channel Request (Web UI / API)
  → Request Queue (PostgreSQL)
  → ACE-Step v1.5 Worker (Apple Silicon MPS, music generation)
  → Generated Track Storage (filesystem + metadata in DB)
  → Streaming Server (Icecast or HLS)
  → Web Player (React SPA with channel selector)
  → Caddy (reverse proxy, port 3200)
```

## Tech Stack
- **Music Generation**: ACE-Step v1.5 (Python, Apple Silicon MPS backend)
- **Backend API**: Python FastAPI
- **Queue**: PostgreSQL-backed job queue (or Redis if needed)
- **Streaming**: Icecast2 or HLS via nginx
- **Frontend**: React with Vite (player UI)
- **Database**: PostgreSQL 16
- **Reverse Proxy**: Caddy
- **All containerized** with Docker Compose

## Key Technical Considerations

### ACE-Step v1.5
- GitHub: https://github.com/ace-step/ACE-Step
- Runs on Apple Silicon via MPS (Metal Performance Shaders)
- The generation worker may need to run outside Docker on the host Mac mini for MPS access
- Docker Compose should handle this via `network_mode: host` or a separate entry point
- If MPS is unavailable in Docker, design the worker as a standalone Python process that connects to the same PostgreSQL

### Streaming
- Icecast2 is well-suited for radio-style streaming
- HLS is more browser-friendly but adds latency
- Consider starting with Icecast2 + a web player that uses an HTML5 audio element
